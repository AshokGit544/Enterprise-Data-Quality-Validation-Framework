# Enterprise Data Quality Validation Framework using Python and Lakehouse Quality Checks

## Project Overview
This project simulates how enterprise data quality checks can be built to improve trust in a data platform.

It is based on a quality framework where data is validated before it is trusted by business users, reports, downstream analytics, or AI and ML use cases.

The project includes:
- schema validation
- freshness validation
- duplicate detection
- null handling checks
- source-to-target reconciliation
- quality reporting outputs

## Business Goal
The goal of this project is to show how data quality checks increase trust in the platform by catching issues early before they affect reports, analytics, or downstream models.

## Tools Used
- Python
- Pandas
- NumPy
- Google Colab
